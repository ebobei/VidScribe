# VidScribe / ВидСкрайб

**VidScribe** — локальная CLI-утилита для сбора транскриптов YouTube-видео по поисковому запросу.

Проще говоря: ты задаёшь тему исследования, VidScribe находит подходящие YouTube-видео, скачивает **только субтитры**, очищает текст, режет его на удобные части и собирает архив, который можно загрузить в AI для дальнейшего анализа.

Проект полезен, когда нужно быстро понять, что говорят в нескольких видео по одной теме: какие мнения повторяются, какие проблемы называют, какие советы дают и какие выводы можно применить на практике.

---

## Что умеет VidScribe

VidScribe умеет:

- искать YouTube-видео по текстовому запросу;
- получать публичную информацию о найденных видео;
- фильтровать неподходящие видео;
- скачивать только субтитры, без видео и аудио;
- выбирать лучшую доступную дорожку субтитров;
- очищать `.vtt` / `.srt` в обычный текст;
- сохранять очищенные транскрипты в `transcripts_clean/`;
- создавать чанки в `chunks/chunks.jsonl`;
- собирать `summary_input.md` для загрузки в AI;
- собирать итоговый архив `research_pack.zip`;
- запускаться в стабильном ночном режиме с SQLite-state, паузами, retry и resume.

---

## Что проект не делает

VidScribe не скачивает видео и аудио.

Также в MVP нет:

- веб-интерфейса;
- сервера;
- Redis;
- базы данных как отдельного сервиса;
- личного кабинета;
- распознавания аудио через Whisper;
- embeddings;
- vector DB;
- полноценной RAG-платформы.

SQLite используется только как локальный файл состояния для стабильного режима. Это не серверная БД.

## Основные понятия

### Что такое `yt-dlp`

`yt-dlp` — это сторонняя консольная утилита для работы с видео-платформами, включая YouTube.

Внутри VidScribe она используется для трёх задач:

1. найти видео по поисковому запросу;
2. получить публичную информацию о видео: название, канал, длительность, ссылку;
3. скачать subtitle-файл, если у видео есть подходящие субтитры.

VidScribe использует `yt-dlp` только в безопасном для проекта режиме: **видео и аудио не скачиваются**. При скачивании субтитров используется режим `--skip-download`.

### Видео-кандидат

Видео-кандидат — это видео, которое VidScribe нашёл по поисковому запросу.

Не каждый кандидат попадёт в итоговый результат. Сначала VidScribe собирает список кандидатов, а потом отбрасывает неподходящие видео.

Например, если указано:

```yaml
candidate_pool_size: 60
```

VidScribe попробует найти до 60 видео-кандидатов.

### Обработанное видео

Обработанное видео — это видео, которое:

1. найдено по запросу;
2. прошло фильтры;
3. имеет подходящие субтитры;
4. получило очищенный транскрипт;
5. попало в итоговый research pack.

Если видео найдено, но у него нет подходящих субтитров, оно будет пропущено.

### Shorts

Shorts — это короткие вертикальные YouTube-видео, похожие на TikTok или Reels.

Обычно они слишком короткие для нормального исследования: в них мало текста, мало контекста и часто нет полезных подробностей. Поэтому VidScribe может пропускать такие видео.

### Live / upcoming

`live` — это прямой эфир.

`upcoming` — это запланированная трансляция, которая ещё не прошла.

VidScribe пропускает такие видео, потому что у них может не быть готовых субтитров или нормального завершённого контента.

### Ручные и автоматические субтитры

**Ручные субтитры** — это субтитры, которые добавил автор видео или редактор. Обычно они качественнее.

**Автоматические субтитры** — это субтитры, которые YouTube сгенерировал сам через распознавание речи. Они могут содержать ошибки, но часто это единственный доступный вариант.

### HTTP 429 / rate limit

HTTP 429 означает, что YouTube временно ограничил частоту запросов.

В обычном режиме такая ошибка может привести к пропуску или падению части обработки. В стабильном режиме VidScribe ждёт 15 минут и пробует снова. Если попытки закончились, конкретное видео помечается как `failed / rate_limited`, а сбор продолжается дальше.

## Установка

Пример для Windows PowerShell:

```powershell
cd path\to\VidScribe

python -m venv .venv
.\.venv\Scripts\Activate.ps1

python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Проверить, что CLI доступен:

```powershell
python -m app.main --help
```

---

## Первый запуск

Для обычного быстрого запуска:

```powershell
python -m app.main collect --config ./requests/example_research.yaml
```

Для быстрой проверки лучше начать с маленького лимита:

```powershell
python -m app.main collect `
  --config ./requests/example_research.yaml `
  --limit 3 `
  --candidate-pool-size 10
```

Пример с сортировкой по дате:

```powershell
python -m app.main collect `
  --config ./requests/example_research.yaml `
  --limit 3 `
  --candidate-pool-size 10 `
  --order date
```

Лучше всего сейчас поддерживаются `relevance` и `date`.

---

## Стабильный ночной режим

Если нужно запустить большой сбор и уйти, используй стабильный режим:

```powershell
python -m app.main collect `
  --config ./requests/private_research.yaml `
  --stable
```

В стабильном режиме VidScribe:

- создаёт `state.sqlite` в папке запуска;
- записывает найденные видео в локальную очередь;
- обрабатывает видео по одному;
- делает паузу между видео;
- при HTTP 429 ждёт 15 минут и пробует снова;
- сохраняет состояние после каждой попытки;
- позволяет продолжить запуск после сбоя;
- собирает частичный research pack из того, что удалось обработать.

Это режим для сценария: **запустил вечером → ушёл спать → утром забрал результат**.

### Продолжить прерванный запуск

Если процесс остановился, можно продолжить по существующей папке запуска:

```powershell
python -m app.main resume --run-dir ./output/example_research/example_research_2026-05-22_2300
```

### Собрать архив из уже обработанных результатов

Если нужно просто пересобрать `videos.csv`, `summary_input.md`, `manifest.json` и `research_pack.zip` из текущего состояния:

```powershell
python -m app.main pack --run-dir ./output/example_research/example_research_2026-05-22_2300
```

Это полезно, если ночью обработалась только часть видео, но уже хочется получить архив по тому, что есть.

## Конфиги запросов

Конфиги лежат в папке `requests/`.

Для нового исследования проще скопировать существующий YAML-файл и изменить настройки под свою тему.

Если репозиторий публичный, лучше держать реальные личные конфиги локально и не коммитить папку `requests/`.

Пример нейтрального конфига:

```yaml
project_name: "example_research"

query: "indoor camera review two way audio"

limit: 20
candidate_pool_size: 60

duration:
  min_seconds: 240
  max_seconds: 3600

languages:
  preferred:
    - "ru"
    - "en"
  allow_auto_subtitles: true
  allow_manual_subtitles: true

youtube:
  order: "relevance"
  region_code: "RU"
  relevance_language: "ru"
  exclude_shorts: true
  exclude_live: true

output:
  directory: "./output/example_research"
  build_zip: true
  build_summary_md: true
  build_chunks_jsonl: true

safety:
  download_video: false
  download_audio: false
  public_dataset: false

stability:
  enabled: true
  max_attempts_per_video: 3
  video_sleep_min_seconds: 120
  video_sleep_max_seconds: 180
  rate_limit_backoff_seconds: 900
  save_partial_results: true
  build_pack_on_partial_success: true
```

## Что означают настройки YAML

### `project_name`

```yaml
project_name: "example_research"
```

Техническое имя исследования.

Используется для названия папки с результатами внутри `output/`.

Лучше писать коротко, латиницей, без пробелов:

```yaml
project_name: "example_research"
project_name: "device_review"
project_name: "tutorial_analysis"
```

Не стоит указывать здесь личные данные, реальные адреса, имена клиентов или внутренние названия рабочих проектов, если репозиторий публичный.

### `query`

```yaml
query: "indoor camera review two way audio"
```

Поисковый запрос для YouTube.

Это то, что VidScribe будет искать.

Хороший запрос обычно состоит из нескольких ключевых слов. Слишком общий запрос даст много мусора.

Для технических тем часто лучше работают английские запросы, потому что по ним больше видео и больше доступных субтитров.

### `limit`

```yaml
limit: 20
```

Максимальное количество видео, которые VidScribe попытается обработать в итоговый research pack.

Важно: `limit: 20` не гарантирует, что в результате будет ровно 20 готовых транскриптов. Часть видео может быть пропущена.

Рекомендации:

```yaml
limit: 3   # быстрый тест
limit: 10  # обычное исследование
limit: 20  # более широкий сбор
```

Для значений сильно выше 20 лучше использовать `--stable`.

### `candidate_pool_size`

```yaml
candidate_pool_size: 60
```

Количество видео-кандидатов, которые VidScribe сначала берёт из поиска.

Это значение обычно должно быть больше, чем `limit`.

Пример:

```yaml
limit: 20
candidate_pool_size: 60
```

Это значит:

```text
найти до 60 видео-кандидатов,
отфильтровать неподходящие,
попробовать получить до 20 обработанных видео.
```

Практическое правило:

```text
candidate_pool_size ≈ limit * 3
```

Для больших значений лучше использовать стабильный режим.

### `duration.min_seconds`

```yaml
duration:
  min_seconds: 240
```

Минимальная длительность видео в секундах.

`240` означает: не брать видео короче 4 минут.

Это помогает отсечь слишком короткие ролики и Shorts.

### `duration.max_seconds`

```yaml
duration:
  max_seconds: 3600
```

Максимальная длительность видео в секундах.

`3600` означает: не брать видео длиннее 1 часа.

Это помогает не собирать многочасовые эфиры, стримы и конференции.

### `languages.preferred`

```yaml
languages:
  preferred:
    - "ru"
    - "en"
```

Список предпочитаемых языков субтитров.

Порядок важен: сначала VidScribe ищет субтитры на первом языке, потом на втором.

Важно: VidScribe не переводит видео. Он использует только те субтитры, которые уже есть у видео.

### `allow_auto_subtitles`

```yaml
allow_auto_subtitles: true
```

Разрешает использовать автоматические субтитры YouTube.

Для большинства исследований лучше оставить `true`, потому что у многих видео нет ручных субтитров.

### `allow_manual_subtitles`

```yaml
allow_manual_subtitles: true
```

Разрешает использовать ручные субтитры.

Обычно лучше оставить `true`.

Если разрешены и ручные, и автоматические субтитры, VidScribe выбирает лучшую дорожку по приоритету:

```text
1. ручные субтитры на первом предпочитаемом языке
2. автоматические субтитры на первом предпочитаемом языке
3. ручные субтитры на втором предпочитаемом языке
4. автоматические субтитры на втором предпочитаемом языке
```

### `stability.enabled`

```yaml
stability:
  enabled: true
```

Включает настройки стабильного режима в конфиге.

Также стабильный режим можно включить флагом CLI:

```powershell
python -m app.main collect --config ./requests/private_research.yaml --stable
```

### `stability.max_attempts_per_video`

```yaml
max_attempts_per_video: 3
```

Сколько раз VidScribe будет пытаться обработать одно видео, если возникают ошибки.

### `stability.video_sleep_min_seconds` / `video_sleep_max_seconds`

```yaml
video_sleep_min_seconds: 120
video_sleep_max_seconds: 180
```

Пауза между обработкой видео.

В этом примере VidScribe ждёт случайное время от 2 до 3 минут между видео. Это снижает вероятность временных ограничений со стороны YouTube.

### `stability.rate_limit_backoff_seconds`

```yaml
rate_limit_backoff_seconds: 900
```

Пауза после HTTP 429 / rate limit.

`900` секунд = 15 минут.

Если YouTube временно ограничил запросы, VidScribe ждёт 15 минут и пробует это же видео снова. Если попытки закончились, видео получает статус `failed / rate_limited`, а сбор продолжается.

## Что появится после запуска

После запуска VidScribe создаёт отдельную папку в `output/`.

Пример:

```text
output/example_research/example_research_2026-05-22_1840/
  state.sqlite
  run.json
  manifest.json
  videos.csv
  summary_input.md
  research_pack.zip

  metadata/
    001__VIDEO_ID.json

  subtitles_raw/
    001__VIDEO_ID.ru.manual.vtt

  transcripts_clean/
    001__VIDEO_ID.ru.manual.txt

  chunks/
    chunks.jsonl

  logs/
    collect.log
```

Главные файлы:

- `summary_input.md` — файл, который удобно загрузить в AI;
- `research_pack.zip` — архив со всеми результатами запуска;
- `videos.csv` — таблица видео, статусов и причин пропуска;
- `state.sqlite` — локальное состояние стабильного запуска;
- `transcripts_clean/*.txt` — очищенные транскрипты;
- `chunks/chunks.jsonl` — чанки для дальнейшей обработки;
- `manifest.json` — техническое описание пакета;
- `run.json` — параметры конкретного запуска.

## Как использовать результат

Обычно сценарий такой:

1. Запустить VidScribe по нужному YAML-конфигу.
2. Открыть папку результата в `output/`.
3. Взять `summary_input.md` или `research_pack.zip`.
4. Загрузить файл в AI.
5. Попросить проанализировать:
   - какие мнения повторяются;
   - какие проблемы чаще всего называют;
   - какие выводы полезны для проекта;
   - какие видео были наиболее информативными;
   - что стоит проверить отдельно.

`summary_input.md` специально сделан человекочитаемым: его можно открыть, быстро просмотреть и понять, что именно было собрано.

## Статусы в `videos.csv`

В таблице `videos.csv` видно, что произошло с каждым видео.

Основные статусы:

- `processed` — видео прошло фильтры, субтитры скачаны, текст очищен, чанки построены;
- `skipped / no_subtitles` — подходящих субтитров нет;
- `skipped / duration_too_short` — видео слишком короткое;
- `skipped / duration_too_long` — видео слишком длинное;
- `skipped / live_video` — прямой эфир или запланированная трансляция;
- `skipped / shorts` — видео похоже на YouTube Shorts;
- `skipped / duplicate` — дубль уже найденного видео;
- `failed / yt_dlp_error` — не удалось скачать subtitle-файл;
- `failed / subtitle_cleaning_error` — subtitle-файл скачан, но не удалось очистить его в текст;
- `failed / rate_limited` — YouTube временно ограничил запросы, а попытки по видео закончились.

---

## Что лежит в `research_pack.zip`

Архив содержит результаты конкретного запуска:

- `run.json`;
- `manifest.json`;
- `videos.csv`;
- `summary_input.md`;
- `metadata/`;
- `subtitles_raw/`;
- `transcripts_clean/`;
- `chunks/`;
- `logs/`.

Архив не включает сам себя и временные директории.

## Ограничения и возможные проблемы

### YouTube может временно ограничивать запросы

Иногда YouTube может временно ограничить частоту запросов. В логах или статусах это может выглядеть как HTTP 429 или ошибка получения данных.

Что можно сделать:

- использовать `--stable`;
- уменьшить `limit`;
- уменьшить `candidate_pool_size`;
- повторить запуск позже;
- не запускать много больших исследований подряд.

### Не у всех видео есть субтитры

Если у видео нет подходящих ручных или автоматических субтитров, оно будет пропущено со статусом:

```text
skipped / no_subtitles
```

Это нормальное поведение.

### Таймкоды в чанках best-effort

`chunks.jsonl` пытается сохранить `start_time` и `end_time` из raw subtitle-файла. Если таймкоды не удалось надёжно сопоставить, поля могут быть `null`.
